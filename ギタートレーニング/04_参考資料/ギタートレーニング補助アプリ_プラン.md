# ギタートレーニング補助アプリ プラン
## コードトーン・トライアド習得支援アプリ

### アプリの目的
ギタリストがコードトーンとトライアドを理解し、Matt Schofieldスタイルのフレージングを習得するための補助ツール

---

## コア機能（優先度順）

### 1. コードトーン・トライアドビューアー（最優先）

#### 機能詳細
**指板図での視覚化**
- 選択したコードの構成音を指板図上で表示
- 色分け表示:
  - ルート: 赤
  - 3rd: 青
  - 5th: 緑
  - 7th: 黄
  - テンション（9th、11th、13th）: 紫

**表示モード**
- **トライアドモード**: ルート、3rd、5thのみ表示
- **セブンスモード**: ルート、3rd、5th、7thを表示
- **テンション含むモード**: 9th、11th、13thも表示

**インタラクティブ機能**
- 指板をタップ/クリックすると、その音を再生
- コードトーンをタップすると、コード全体の響きを再生
- 各コードトーンの音名を表示（A, C#, E, Gなど）

**ポジション表示**
- 複数のポジションを表示（開放弦付近、3フレット付近、5フレット付近など）
- ポジション切り替えボタン
- 全ポジションを一度に表示するモード

#### UIイメージ
```
┌─────────────────────────────┐
│ コード選択: [A7 ▼]          │
│                             │
│  [トライアド] [セブンス]     │
│  [テンション含む]           │
│                             │
│  ┌─────────────────────┐   │
│  │  指板図（6弦表示）  │   │
│  │  ●(赤) = ルート     │   │
│  │  ●(青) = 3rd        │   │
│  │  ●(緑) = 5th        │   │
│  │  ●(黄) = 7th        │   │
│  └─────────────────────┘   │
│                             │
│  構成音: A, C#, E, G        │
│  [再生] [ポジション切替]    │
└─────────────────────────────┘
```

### 2. コード進行ビューアー（最優先）

#### 機能詳細
**コード進行の設定**
- プリセット進行:
  - 12小節ブルース（I-IV-V）
  - ジャズブルース
  - ツー・ファイブ・ワン（ii-V-I）
  - カスタム進行（ユーザー入力）

**各コードのコードトーン表示**
- 進行中の各コードのコードトーンを指板図で表示
- 現在のコードをハイライト
- 次のコードを予告表示

**再生機能**
- バッキングトラック再生
- コードトーンを音で確認
- ループ再生
- 速度調整（BPM）

**練習モード**
- **コードトーン着地練習**: 各コードの小節で、コードトーンに着地する練習
- **アルペジオ練習**: 各コードのコードトーンをアルペジオで弾く練習

#### UIイメージ
```
┌─────────────────────────────┐
│ 進行: [12小節ブルース ▼]    │
│                             │
│  A7 ── D7 ── A7 ── A7      │
│  ●     ●     ●     ●        │
│  (現在)                     │
│                             │
│  ┌─────────────────────┐   │
│  │  A7のコードトーン   │   │
│  │  指板図表示         │   │
│  └─────────────────────┘   │
│                             │
│  [再生] [停止] [ループ]     │
│  BPM: [120] [速度調整]      │
│                             │
│  [コードトーン着地練習]     │
│  [アルペジオ練習]           │
└─────────────────────────────┘
```

### 3. スケールとコードトーンの関係表示（高優先度）

#### 機能詳細
**スケール選択**
- メジャースケール
- マイナースケール
- ドリアン
- ミクソリディアン
- ブルーススケール
- ペンタトニック

**コードトーンのハイライト**
- 選択したスケールを指板図で表示
- 現在のコードのコードトーンを色分けしてハイライト
- コードトーンとスケールの関係を視覚的に理解

**コード進行モード**
- コード進行を設定すると、各コードに適したスケールを自動提案
- スケールチェンジのタイミングを表示

#### UIイメージ

### 4. トライアド練習ツール（高優先度）

#### 機能詳細
**練習モード**
- **ポジション練習**: 指定したポジションでトライアドを練習
- **キー練習**: 指定したキーで、全トライアドを練習
- **ランダム練習**: ランダムにトライアドを表示、ユーザーが弾く

**フィードバック**
- ユーザーが弾いた音を検出（オプション）
- 正解/不正解の表示
- スコア表示

**練習内容**
- メジャートライアド（全キー）
- マイナートライアド（全キー）
- 転回形の練習

### 5. コードトーン着地練習ツール（高優先度）

#### 機能詳細
**練習モード**
- バッキングトラックに合わせて、各コードのコードトーンに着地する練習
- コード進行を設定
- 各小節で、コードトーンに着地するタイミングを表示

**フィードバック**
- ユーザーが弾いた音を検出（オプション）
- コードトーンに着地できたか判定
- スコア表示

**難易度設定**
- **初級**: 1小節1音（ルートのみ）
- **中級**: 1小節2-3音（ルート、3rd、5th）
- **上級**: 自由にフレーズを展開し、コードトーンに着地

#### UIイメージ
```
┌─────────────────────────────┐
│ コードトーン着地練習        │
│                             │
│  進行: A7 → D7 → A7...     │
│  現在: A7 (小節1/4)         │
│                             │
│  ┌─────────────────────┐   │
│  │  指板図             │   │
│  │  着地目標: A(ルート)│   │
│  │  ●(赤) = 着地目標   │   │
│  └─────────────────────┘   │
│                             │
│  [再生] [停止]              │
│  スコア: 8/10               │
│  難易度: [中級 ▼]          │
└─────────────────────────────┘
```

---

### 6. バッキングトラック生成（中優先度）

#### 機能詳細
**トラック生成**
- コード進行を入力すると、自動でバッキングトラックを生成
- スタイル選択:
  - ブルース
  - ジャズ
  - ロック
  - カスタム

**再生機能**
- ループ再生
- 速度調整（BPM）
- キー変更
- 各楽器の音量調整（ドラム、ベース、ギターなど）

**保存機能**
- お気に入りの進行を保存
- カスタム進行の保存

---

### 7. 練習ログ・進捗管理（中優先度）

#### 機能詳細
**練習記録**
- 練習日時、時間、内容を記録
- 練習したコード、スケール、進行を記録
- メモ機能

**進捗表示**
- グラフ表示（練習時間、練習内容）
- 習得状況（コードトーン、トライアドの理解度）
- 目標設定と達成状況

**統計情報**
- 最も練習したコード
- 最も練習した進行
- 練習時間の推移

---

### 8. フレーズ分析ツール（低優先度・将来実装）

#### 機能詳細
**フレーズ入力**
- タブ譜形式でフレーズを入力
- または、音声認識でフレーズを入力

**分析機能**
- 入力したフレーズに含まれるコードトーンを分析
- コードトーンが適切に使われているか判定
- 改善提案

**Matt Schofieldスタイル分析**
- 実際のMatt Schofieldのフレーズを分析
- コードトーンの使い方を可視化

## 技術仕様（簡易版）

### フロントエンド
- **フレームワーク**: React / Next.js または React Native（モバイル）
- **指板図レンダリング**: SVG または Canvas
- **音声再生**: Web Audio API（Web）、react-native-sound（モバイル）
- **音声認識**: Web Speech API（Web、オプション）

### バックエンド（必要に応じて）
- **バッキングトラック生成**: サーバー側でMIDI生成、またはクライアント側でTone.js使用
- **データ保存**: ローカルストレージ（Web）、SQLite（モバイル）

### データ構造
```typescript
// コードトーンのデータ構造
interface ChordTone {
  root: string;        // "A"
  third: string;       // "C#"
  fifth: string;       // "E"
  seventh?: string;    // "G"
  tensions?: string[]; // ["B", "D", "F#"]
}

// 指板位置
interface FretPosition {
  string: number;      // 1-6
  fret: number;        // 0-24
  note: string;        // "A", "C#", etc.
}
```

## UI/UX設計のポイント

### デザイン原則
1. **視覚的で直感的**: 指板図を中心に、コードトーンを色分けして表示
2. **シンプル**: 複雑な機能を隠し、必要に応じて表示
3. **レスポンシブ**: スマホ、タブレット、PCで使いやすい
4. **ダークモード**: 長時間使用時の目の負担軽減

### 主要画面構成
1. **ホーム画面**: クイックアクセス、最近の練習
2. **コードトーンビューアー**: 指板図表示、コード選択
3. **コード進行ビューアー**: 進行設定、コードトーン表示
4. **練習ツール**: トライアド練習、コードトーン着地練習
5. **練習ログ**: 記録一覧、進捗表示

## 成功指標

### ユーザー体験
- コードトーンが視覚的に理解できる
- トライアドの練習が効率的にできる
- コード進行に合わせたコードトーンの把握ができる

### 技術指標
- 指板図の描画がスムーズ（60fps）
- 音声再生の遅延が少ない（100ms以下）
- レスポンシブデザインが適切に機能

---

## 次のステップ

1. **プロトタイプ作成**: コードトーンビューアーのプロトタイプを作成
2. **技術検証**: 指板図のレンダリング、音声再生の実現可能性確認
3. **UI/UXデザイン**: 主要画面のデザイン作成
4. **開発開始**: MVPから段階的に開発
